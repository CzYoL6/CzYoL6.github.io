Github重要概念

Fork

将别人的项目复制克隆到自己的仓库，与原仓库独立

Pull request

对克隆的仓库作出修改后，向原仓库发起pull request，原作者查看后感觉不错，可以选择合并。

Watch

关注仓库，更新时会接收到通知

Issue

发现bug，发起讨论

Github开源项目贡献流程

  • Fork仓库
  • 在forked仓库中修改
  • 提交pull request
  • 等待作者操作

Git常用操作

  • git config --global user.name "name"
    配置用户名

  • git config --global user.email "email"
    配置用户邮箱

  • git config --list
    查看git配置信息

  • git init

初始化本地git仓库

  • git log

查看本地日志，即当前分支的历史版本

  • git reflog

查看远端日志，即

  • git add name

添加名为name的文件到暂存区

  • git add .

添加所有文件到暂存区

  • git rm name

从暂存区删除名为name的文件

  • git commit -m "string"

将本地修改过的文件提交到本地仓库

  • git push

将本地仓库的更新发送到远端

  • git push --set-upstream origin <name>

将name分支(本地新切的分支)的上流分支设置为远端的分支(origin)

  • git pull

拉取最新的commit = fetch + 自动merge

  • git status

查看所有文件的状态

  • git status -s

查看所有文件的状态（以简洁的方式）

  • git branch

查看所有本地分支

  • git merge <branchName>

合并branchName分支变更到当前分支

  • git fetch

拉取远端仓库分支

  • git clone url

克隆仓库到本地

  • git checkout -b <name> <template>

创建新的分支(name:新分支的名字，template:以哪个分支或commit 为模板，缺省默认为当前分支) 新建的分支会继承模板的commit记录，而后便与之独立

  • git checkout <name>

切换到name分支

  • git checkout develop

切换到develop分支

  • git checkout master

切换到master分支

  • git reset id --hard

不保存所有变更

  • git reset id --soft

保留变更且变更内容处于staged

  • git reset id --mixed

保留变更且变更内容处于modified（缺省默认）

  • git rm --cached index.html

    删除缓存中的文件
