RESTful API with Nodejs & Express

含义及特点

REST = Representational State Transfer

RESTFUL特点包括：

  • 每一个URI代表1种资源；

  • 客户端使用GET、POST、PUT、DELETE4个表示操作方式的动词对服务端资源进行操作：GET用来获取资源，POST用来新建资源（也可以用于更新资源），PUT用来更新资源，DELETE用来删除资源；

  • 通过操作资源的表现形式来操作资源；

  • 资源的表现形式是XML或者HTML；

  • 客户端与服务端之间的交互在请求之间是无状态的，从客户端到服务端的每个请求都必须包含理解请求所必需的信息。

CRUD = Create Read Update Delete

Http Request

  • GET: 获取资源
  • PUT: 更新资源
  • DELETE: 删除资源
  • POST: 新建资源

什么是node.js？

Node.js is a JavaScript runtime built on Chrome’s V8 JavaScript engine.（基于Chrome的V8 JavaScript引擎构建的JavaScript运行时。）

Module

创建Module

  • 每个js文件都是一个module。

其中的变量和函数对于其他module都是不可见的。

可以通过module.exports.xx = xxx;来将其变为public。

加载Module——require

  • 用module名代表一个函数

Module Wrapper Function

nodejs中的module其实被一个module wrapper function包裹，参数列表中的参数包含经常使用的：

  • exports: module.exports的引用，不能直接赋值（如exports = function(){}），因为exports一直代表module.exports。但是可以使用exports.log = log;。
  • require: 加载module
  • module: 该模块
  • __filename: 该模块的文件名

平衡二叉树

平衡二叉树的性质

  • 对于任意结点，其左右子树必为平衡二叉树，且左右子树的高度差的绝对值$\leq 1$。
  • 插入，查找，删除的时间复杂度最好情况和最坏情况都维持在$O(logN)$。

几种操作

左旋

右旋

失衡的调整

LL型调整

当一个子树因为在其左子结点的左子树插入结点而失衡时，右旋根节点

RR型调整

当一个子树因为在其右子节点的右子树插入结点而失衡时，左旋根节点

LR型调整

当一个子树因为在其左子结点的右子树插入结点而失衡时，左旋左子结点，再右旋根节点

RL型调整

当一个子树因为在其右子节点的左子树插入结点而失衡时，右旋右子结点，再左旋根节点

红黑树

红黑树的性质

(添加外结点，将外结点null视作叶子结点。)

红黑树是一颗满足下列性质的二叉搜索树：

  1. 每个结点或是红色的，或是黑色的。

  2. 根结点时黑色的。

  3. 每个叶结点(NULL)是黑色的。

  4. 如果一个结点是红色的，则它的两个子结点都是黑色的。

  5. 对每个结点，从结点到其所有后代叶结点的简单路径上，均包含相同数目的黑色结点。

这些性质通过对任意一条从根到叶子的简单路径上各个结点的颜色进行约束，从而确保没有一条路径会比其他路径长出2倍，因而是近似平衡的。

红黑树的基本操作

红黑树的插入、删除、查找等操作均与二叉搜索树相同。

红黑树的调整操作

为了维持红黑树的性质，在每一步操作过后均需进行调整。

插入

新插入的结点x默认设为红色，其两个叶子节点为黑色。

  • 若x的父节点为黑色，则无需调整。可以看作x的黑叶子节点替换了原来父节点的黑叶子节点，因此路径上的黑色结点数不会变化。

  • 若x的父节点为红色，由于x为红色，因此违反了性质4。

传送门

分析：

记录最短值和次短值，每次dijkstra更新。
注意：

  • 如果用从que中取出的值更新了最短值，那么就将最短值放入que中。
  • 如果用从que中取出的值更新了次短值，那么就将次短值放入que中。
  • 从que中取出值后，设取出的最短/次短值为dist，那么在后续更新中写成$dis[v] = dist + len$而非$dis[v] = dis[u] + len.$，因为无法判断是该值u的最短值或次短值（也可以在状态中加入最短和次短的标记。）
